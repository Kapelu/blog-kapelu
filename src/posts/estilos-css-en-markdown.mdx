# Base de marcado de Tailwind

**Tailwind Markdown Base** es un complemento simple que agrega estilo base para elementos generados por Markdown o un wysiwyg desde un cms.

## Primeros pasos

### Instalación
```sh
    # Install with NPM
    npm install --save @geoffcodesthings/tailwind-md-base

    # Or use Yarn
    yarn add @geoffcodesthings/tailwind-md-base
```

### Uso
```jsx
    // tailwind.config.js
    const tailwindMdBase = require('@geoffcodesthings/tailwind-md-base');

    module.exports = {
      theme: {
        ...
      },
      plugins: [tailwindMdBase()],
    };
```
------------------------------------------------------------------------

## Configuración

### Clase de wrapper

De forma predeterminada, Tailwind Markdown Base envuelve sus estilos generados en una clase`.markdown`. Esto, por supuesto, se puede configurar en`tailwind.config.js`:

```jsx
    // tailwind.config.js
    const tailwindMdBase = require('@geoffcodesthings/tailwind-md-base');

    module.exports = {
      theme: {
        markdownBase: {
          wrapperClass: 'content',
        },
      },
      plugins: [tailwindMdBase()],
    };
```

> El ejemplo anterior envolvería los estilos generados por el complemento`.content` en lugar del valor predeterminado`.markdown`.

### Configuración predeterminada

Con la excepción de`wrapperClass` , la configuración predeterminada es simplemente la sintaxis CSS-in-JS. Esto le permite personalizar los estilos tanto como necesite de manera estandarizada.

### Estilos adicionales

Actualmente solo tenemos estilos predeterminados para los elementos más comunes generados por Markdown. Sin embargo, debido a la sintaxis CSS-in-JS, puede agregar estilo para cualquier elemento en su configuración.

Por ejemplo, si desea diseñar`img` elementos dentro del espacio de`.markdown` nombres, puede hacerlo de la siguiente manera:

```jsx
    // tailwind.config.js
    const defaultTheme = require('tailwindcss/defaultTheme');
    const tailwindMdBase = require('@geoffcodesthings/tailwind-md-base');

    module.exports = {
      theme: {
        markdownBase: {
          img: {
            maxWidth: '100%',
            borderWidth: defaultTheme.borderWidth.default,
            borderColor: defaultTheme.colors.gray[600],
          },
        },
      },
      plugins: [tailwindMdBase()],
    };
```

El ejemplo anterior agregaría lo siguiente al CSS generado:

```css
    .markdown > img {
      max-width: 100%;
      border-width: 1px;
      border-color: #718096;
    }
```

### Párrafos

Los párrafos reciben`1rem` de`margin-bottom` para proporcionar cierto espacio entre ellos.

Vivamus sagittis lacus vel augue laoreet rutrum faucibus dolor auctor. Duis mollis, est non commodo luctus.

``` html
<p>Vivamus sagittis lacus vel augue laoreet rutrum faucibus dolor auctor. Duis mollis, est non commodo luctus.</p>
```

El CSS generado para los párrafos por este complemento es:

```css
    .markdown p {

    }
```

### Código

`code`La etiqueta está diseñada simplemente con la intención de usarse con`inline code` . También se puede utilizar con`code blocks` resaltado de sintaxis si no se utiliza.

Así es como se`inline code` ve.

``` html
<p>Así es como se <code>inline code</code> ve.</p>
```

El CSS generado para`code` las etiquetas por este complemento es:

```css
    .markdown code {
      background-color: #e2e8f0;

      padding-right: 0.5rem;

      padding-bottom: 1px;
```

### Listas

Listas,`ul` y`ol` se les quita su estilo en Tailwind. Este complemento agrega algunos de esos estilos nuevamente.

- Manzanas
- Naranjas
- Plátanos

1.  Manzanas
2.  Naranjas
3.  Plátanos

``` html
<!-- Lista desordenada -->
<ul>
  <li>Apples</li>
  <li>Oranges</li>
  <li>Bananas</li>
</ul>
<!-- lista ordenada -->
<ol>
  <li>Apples</li>
  <li>Oranges</li>
  <li>Bananas</li>
</ol>
```

El CSS generado para las listas por este complemento es:

```css
    .markdown ul {
      list-style-type: disc;
      list-style-position: inside;
      margin-top: 1rem;
      margin-bottom: 1rem;
    }

    .markdown ol {
      list-style-type: decimal;
      list-style-position: inside:
      margin-top: 1rem;
      margin-bottom: 1rem;
    }
```
